# Editorial Notes

Hand-curated companion to `INDEX.md` (which is auto-regenerated by
`scripts/find-plugins.mjs`). Use this file for things the script can't infer:
who's behind a plugin, why it matters, gotchas, dependencies on internal APIs.

## Watchlist

- **Joost de Valk's plugins** ([jdevalk/emdash-plugin-seo](https://github.com/jdevalk/emdash-plugin-seo),
  [jdevalk/emdash-plugin-lettermint](https://github.com/jdevalk/emdash-plugin-lettermint),
  [jdevalk/skills](https://github.com/jdevalk/skills)) — Yoast founder publishing
  EmDash plugins is the single strongest "this is the WP successor" signal
  in the ecosystem. Watch for new repos under this user.
- **piiiico/emdash-plugins** — Trust suite (commitment-relay, publisher-profile,
  trust-badges) overlaps with our `wellknown` data-sovereignty thesis. Worth
  reviewing as plat-trunk evolves.
- **chrisjohnleah/emdashcms-org** — Community plugin & theme registry
  (not Cloudflare-affiliated). Becomes the canonical discovery surface as the
  ecosystem grows; check it before any custom search.

## Categories the script gets wrong

- The category heuristic is keyword-based; if a plugin's name and description
  are vague, it lands in `other`. Spot-check `INDEX.md` after each run.

## Generation

```sh
mise run plugins:catalog          # default: write INDEX.md + plugins.json + stubs
mise run plugins:catalog:full     # also fetches README excerpts (slower)
mise run plugins:catalog:dry      # print to stdout, write nothing
```

Auth: needs a GitHub token (5000 req/hr authed vs 60 req/hr unauth). With fnox:

```sh
fnox exec -- mise run plugins:catalog
```
